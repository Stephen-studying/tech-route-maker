# Demo Source Brief: Software Architecture Route

This synthetic example shows how `tech-route-maker` can map a software project into an editable engineering architecture route.

## Project Goal

Design a route diagram for a repository that turns user documents into validated analytical reports and dashboards.

## Source Evidence

- Inputs: documents, spreadsheets, PDFs, and user configuration.
- Requirements: upload ingestion, metadata extraction, schema normalization, analysis orchestration, report generation, and review.
- Architecture: ingestion service, parser workers, normalized data store, analysis engine, renderer layer, and artifact registry.
- Quality controls: validation rules, provenance tracking, output snapshots, regression tests, and user review checkpoints.
- Deployment: local development, CI validation, packaged examples, and future hosted preview.
- Output: HTML report, editable charts, dashboard view, Markdown summary, and traceable source metadata.

## User Choices For This Demo

- Figure purpose/subtype: Software architecture route
- Output formats: PPTX, SVG, Draw.io, HTML, Markdown, JSON
- Layout: Engineering architecture
- Visual style: Schematic precision
